Services

What OIDCFlow offers

A complete OIDC identity provider built to the spec — designed to show exactly how authentication works under the hood.

ID

OIDC Identity Provider

Standards-compliant OpenID Connect provider exposing discovery, JWKS, authorization, token, and userinfo endpoints.

⇄

OAuth 2.0 Auth Code Flow

Full Authorization Code Flow for securely issuing tokens to web and mobile apps after user consent.

#

PKCE Protection

Proof Key for Code Exchange (RFC 7636) prevents authorization code interception attacks — required for public clients.

JWT

Signed JWT Tokens

RS256-signed access tokens, ID tokens, and opaque refresh tokens. Public keys published via JWKS for independent verification.

∞

Single Sign-On (SSO)

Users authenticate once with OIDCFlow and seamlessly access all connected applications without re-entering credentials.

↩

Back-channel Logout

Secure server-to-server logout notifications so connected apps can invalidate sessions when a user signs out from any client.

How it works

From registration to tokens

Four steps to integrate OIDCFlow into your application.

1

Register your company

Sign up as a company, receive a unique client ID, and configure your redirect URI and optional back-channel logout endpoint.

2

Redirect to OIDCFlow

When a user wants to log in, redirect them to the authorization endpoint with a PKCE code challenge and your client ID.

3

User authenticates

Users sign in with their OIDCFlow account and grant consent for the requested scopes (openid, profile, email).

4

Exchange for tokens

Exchange the short-lived authorization code for JWT access, ID, and refresh tokens. Call userinfo to get user claims.
